A PowerDraftHighlight:
AccuDraw
Rob Brown

Our biggest challenge while developing PowerDraft was to try to make a product that is easy to learn, yet highly powerful and productive at all experience levels. We wanted to avoid having to trade functionality for simplicity. Instead, we needed comprehensive, yet easy to learn solutions. This meant elegant, universal and extensible tools that allow for progressive learning.

While production drafters are the target customers for PowerDraft, we also wanted to be sure that any solutions we developed were not unnecessarily limited in scope. If we didn't make such considerations from the start, we could have ended up with tools that were limited as 2D drafting tools only. It would be unfair to users to require them to learn a new tool from scratch if they wished to do conceptual design or advanced 3D modeling. Therefore, we strove for solutions that would be just as useful in these contexts, and would, therefore, be appropriate in future versions of MicroStation.

The Challenge of
Drawing on a Computer

At the heart of the interface to any CAD system is precision data point input—the main point of drafting being to draw geometry precisely. On paper, the tools for doing so are T-squares, triangles, compasses and scales (rulers). More recently, drafting machines have consolidated many of these tools into a single entity. On computers, however, there is really no direct equivalent of these tools.

Instead, the physical interface to computers is typically a keyboard and a mouse. Traditionally, CAD systems use the keyboard for precision drawing input, and the mouse for freehand work. Using the mouse is certainly easier, more intuitive and more closely approximates drawing on paper. Furthermore, the feedback afforded by dynamic “rubber banding” makes it quite clear to the user what will result when a data point is entered. The problem is it's not very useful for “real work,” which has to be precise. Working in 3D, the mouse has another limitation: it is inherently a 2D input device.

Most input, therefore, comes from the keyboard. MicroStation users quickly become very familiar with the DL= and DX= keyins, so they can precisely position the points that define their geometry. But “drawing” by keyboard input is hardly drawing at all, it is simply not as interactive and visual as drawing with the mouse. So the need is for a compre-hensive tool allowing keyboard and mouse to work together, without sacrificing the precision, speed and 3D capabilities of keyboard input, or the intuitive nature, interactivity and feedback of mouse drawing.

The Solution: AccuDraw

Enter AccuDraw. As the name implies, this tool provides the accuracy inherent to keyboard input, while still allowing the user to really “draw.” It consolidates, and effectively replaces, the functionality of many existing MicroStation features: the precision input dialog box, unit lock, axis lock, active depth and depth lock, auxiliary plane lock, 3D data point, and so on.
