# OriginMarker

`OriginMarker` is the visual representation of the AccuDraw drawing compass. It displays the current drawing `origin` and the orientation of the active 2D drawing `plane` within the 3D scene, providing essential context for the user.

### Core Philosophy

This class is a direct implementation of the AccuDraw concept. Its purpose is to make the abstract 2D drawing context tangible and intuitive in a 3D world. By showing the user precisely where they are drawing (the origin) and on what surface (the plane's orientation), it removes ambiguity and enables precise construction. The animated transitions for position and rotation are key to helping the user maintain spatial awareness as the context changes.

### Primary API Usage

An `OriginMarker` is typically instantiated once and managed by the `BaseController`. Its visual appearance is updated by calling one of its animated setter methods.

```javascript
// In ControllerSetup.js

// 1. Create the marker, passing the threeDView and options
const originMarker = new OriginMarker(threeDView, {
  squircleAmount: 0.3,
  center: [0, 0, 0],
  color: 0x88ccff,
  size: 0.7,
});

// 2. Add its visual object to the scene
scene.add(originMarker.getObject3D());

// 3. Update its position and rotation (typically from BaseController)
originMarker.setCenterAnimated([10, 5, 0], 0.3); // new center, 0.3s duration
originMarker.setRotationAnimated(newRotationMatrix, 0.3);

// 4. Update other visual properties
originMarker.setSizeAnimated(2.0, 0.5); // new size, 0.5s duration
```

### Key Properties and Components

#### Constructor Options

-   `squircleAmount` (0 to 1): Controls the shape, from a perfect square (0) to a perfect circle (1).
-   `center` (`[x, y, z]`): The 3D coordinate of the marker's origin.
-   `rotationMatrix`: A 3x3 matrix defining the orientation of the marker.
-   `color`: The base color of the marker's fill and outline.
-   `size`: The overall scale of the marker.
-   `cameraOffset`: A small value to push the marker towards the camera, preventing z-fighting with geometry on the same plane.
-   `depthTest` (boolean): Whether the marker should be occluded by other objects.

#### Visual Components

1.  **Main Shape**: A 3D squircle/plane generated by the `Squircle3D` class. It has a semi-transparent fill and an optional outline.
2.  **Decorations**: A group of small meshes that indicate the local X (red) and Y (green) axes of the drawing plane, along with a white sphere at the center.
3.  **Index Indicator**: A white line segment that appears when the user's cursor is snapped (indexed) to one of the marker's axes. It draws a line from the origin to the cursor's projected point on the axis.

### Method Details

-   `getObject3D()`: Returns the `THREE.Group` that contains all visual components of the marker. This is what you add to the scene.
-   `setCenterAnimated(newCenter, duration)`: Smoothly interpolates the marker's position from its current center to `newCenter` over `duration` seconds.
-   `setRotationAnimated(newRotationMatrix, duration)`: Smoothly interpolates the marker's orientation using quaternion slerping (Spherical Linear Interpolation) to get from the current rotation to the new one.
-   `setColorAnimated(newColor, duration)`: Animates the color by interpolating in HSL color space, which produces more visually pleasing transitions (e.g., through a rainbow) than RGB interpolation.
-   `setSquircleAnimated(newSquircle, duration)`: Animates the `squircleAmount` property.
-   `setSizeAnimated(targetSize, duration)`: Animates the `size` property.
-   `update(params)`: A general-purpose method to update any of the constructor options. This is called internally by the animated setters.
-   `updateIndexIndicator(start, end, indexedToAxis)`: Called by `BaseController` during mouse movement. If `indexedToAxis` is true, it makes the indicator line visible and sets its start/end points.

### Implementation Notes

-   **Dependency on `Squircle3D`**: The core plane geometry is not generated by this class directly but is delegated to an instance of `Squircle3D`, which handles the complex SVG path math for creating the shape.
-   `_createDecorations()`: This internal method is responsible for generating the axis tick marks and the center sphere. It rebuilds these decorations whenever the marker's `size` or `rotationMatrix` changes.
-   `_computeTransformMatrix()`: This is a crucial internal utility. It combines the `center`, `rotationMatrix`, and `cameraOffset` properties into a single `THREE.Matrix4` transform. This matrix is then applied to the marker's geometry to position and orient it correctly in the world space. The camera offset logic is handled here by pushing the final position slightly along the vector from the center to the camera.
-   `isAnimating`: This is a counter used as a flag to prevent the index indicator from appearing while the marker itself is in motion, which would be visually distracting.
